# Architectural Decision Records

ADRs for Tribe Y record significant architectural decisions: what was decided, why, and consequences. ADRs are numbered sequentially and never edited once Accepted. To revise an accepted decision, create a new ADR and mark the old one "Superseded by NNNN".

Use `docs/agent/decisions/_template.md` for new ADRs.
